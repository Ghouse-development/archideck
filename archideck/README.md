# ArchiDeck - Gハウス 設計業務管理システム

住宅案件管理とメール発注業務を統合した、設計士・インテリアコーディネーター・外構担当向けの業務効率化システムです。

## 主な機能

### 1. 案件管理（5部構成カード）
- **業務内容**: タスク進捗をチェックボックスで管理
- **タスク**: カスタムタスクの追加・期限管理
- **メモ欄**: IC・外構と共有できるメモ
- **議事録**: ファイルアップロード機能
- **引継書**: 引継ぎ情報の記録

### 2. 担当者管理
- **設計担当**: 設計業務全般
- **IC担当**: インテリアコーディネート業務
- **外構担当**: 外構・エクステリア業務

### 3. サイドバー色分け表示
設計担当の申請GO未完了数に応じた色分け:
- 4以下: 青色
- 5以上: 黄色
- 7以上: 名前が赤色

### 4. 申請GO条件チェック
以下の条件を満たさないと押せない:
- 太陽光依頼が「営業共有済」
- 給排水依頼が「保存済」
- 換気図依頼が「保存済」
- サッシ依頼が「保存済」

### 5. メールテンプレート管理
- 業者別テンプレートの追加・編集・削除
- カテゴリ別管理（設計/IC/外構）
- 変数置換機能

### 6. kintone連携設定
- ドメイン・アプリID・APIトークン設定
- フィールドマッピング
- 接続テスト機能

## セットアップ手順

### ステップ1: Supabaseテーブル作成

1. Supabaseダッシュボードにアクセス
2. SQL Editorを開く
3. `sql/production/supabase_all_in_one.sql` を実行
4. `sql/migrations/v3_archideck_upgrade.sql` を実行

### ステップ2: アプリケーション起動

1. `index.html` をブラウザで開く
2. 初回アクセス時、データが自動的に読み込まれます

## 使い方

### 案件を追加する

1. 📋 案件管理タブを開く
2. 右上の「案件追加」ボタンをクリック
3. 以下の情報を入力:
   - お客様名（必須）
   - 担当設計士（必須）
   - IC担当
   - 外構担当
   - 仕様（LIFE、LIFE+ Limited等）
   - メモ（任意）
4. 「保存」をクリック

### タスクの進捗を更新する

1. 案件カード内のチェックボックスをクリック
2. ステータスをドロップダウンから選択
3. 自動的にSupabaseに保存されます

### 5部構成カードの使い方

1. **業務内容**: チェックボックスとステータスで進捗管理
2. **タスク**: カスタムタスクを追加・期限設定
3. **メモ欄**: 「メモを保存」でIC・外構と共有
4. **議事録**: ファイルをドラッグ&ドロップでアップロード
5. **引継書**: 内容を入力して「引継書を保存」

## データベース構造

### 主要テーブル

1. **designers** - 担当者情報（設計/IC/外構）
2. **projects** - 案件情報
3. **tasks** - タスク定義
4. **vendors_v2** - 業者情報
5. **project_tasks** - 案件タスク（カスタム）
6. **project_minutes** - 議事録
7. **project_handovers** - 引継書
8. **notifications** - 通知
9. **kintone_settings** - kintone連携設定

## 注意事項

### データの安全性
- 全てのデータはSupabaseに自動保存されます
- 定期的にバックアップを推奨します

### ブラウザ互換性
- Chrome、Edge、Safari、Firefox（最新版）で動作確認済み
- インターネット接続が必要です
